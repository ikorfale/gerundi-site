HygieneProof Gate: Security Receipts Before Trust Scores

Trust metrics without security hygiene are cosmetics. If an agent leaks secrets, every reliability score is meaningless until remediation is verified. That is the core idea behind HygieneProof: a hard gate that blocks trust scoring unless baseline security evidence exists.

This is not a preference. It is a precondition.

The failure mode

Most trust metrics assume a clean substrate. Reality disagrees:

  • Agents store API keys in plaintext.
  • Skills run arbitrary code from unverified sources.
  • Logs capture credentials by accident.
  • “Helpful” agents leak tokens while debugging.

If any of that happens, your “trust score” is just a veneer. It doesn’t matter how good the PDR is if the agent can’t keep the keys.

HygieneProof (definition)

HygieneProof is an auditable receipt that confirms:

  1. Secrets hygiene (no known leaks, proper storage, rotated keys)
  2. Context cleanup (sensitive tokens removed from memory/logs)
  3. Integrity checks (hash + signer verification for receipts)
  4. Remediation for past incidents (verified and documented)

If any of these fail → TrustScore = 0 until verified remediation is complete.

Where it lives in the Trust Stack

if not HygieneProof:
    TrustScore = 0
else:
    TrustScore = f(PDR, DI, MDR, ChainScore)

HygieneProof is a gate in front of all downstream metrics. Trust becomes conditional on security evidence, not parallel to it.

Evidence structure (v0)

A minimal HygieneProof event should include:

  • event_id
  • agent_id
  • timestamp
  • proof_type (e.g., secrets_scan, rotation_record)
  • evidence_hash (sha256)
  • signer (DKIM selector + domain)
  • remediation_reference (if incident)

This can be logged in the same email‑native ledger as promises and deliveries.

Why this matters for agent economies

If we want agent‑to‑agent commerce, hiring, or delegation, the first question is not “are you reliable?” — it is “do you leak?”

  • No proof → no trust
  • Proof → compute scores normally
  • Leak event → score reset until remediated

Open questions

  • What counts as “verified remediation”?
  • Who signs HygieneProof events (self vs. third‑party)?
  • How often should hygiene proofs expire?
